Concept Matériel OS & Mods Support

Le Concept

La ROCKNITE CONSOLE est une console portable et modulaire, conçue pour être ouverte, évolutive, et pensée pour les mods. Contrairement aux consoles classiques, elle permet de changer ses composants, de personnaliser son système, et de rester accessible aux bidouilleurs comme aux gamers.

Deux CPU : l'un pour l'interface/menu et la gestion des manettes, l'autre pour les jeux. Résultat ? Si le jeu crash, le menu continue de fonctionner. Fini les freezes comme sur la Switch 1 ou Zorin OS !

Matériel Modulaire

Chaque composant de la console est pensé pour être facilement remplaçable : écran, processeur, mémoire, stockage, manettes détachables... Plus besoin de jeter toute la console pour une mise à niveau.

OS Custom et Architecture

L'OS de la ROCKNITE CONSOLE est entièrement custom, basé sur Ubuntu Server, et développé avec Electron + NodeJS non compilé pour un modding extrême. Tout utilisateur peut modifier, améliorer ou adapter l'interface selon ses besoins sans avoir à recompiler quoi que ce soit. C’est un OS fait pour évoluer avec sa communauté.

Le système est divisé en 3 dossiers principaux :

  • boot/ : vérifie la stabilité du système avant de charger l'OS principal.
  • os1/ : version actuelle de l'OS en service.
  • os2/ : ancienne version de secours. Si os1/os.txt est corrompu ou absent, le boot renomme les dossiers et relance depuis l'ancienne version automatiquement.

Un système robuste qui évite le brick de la console, même après une mise à jour ratée.

Proton + UI Custom sont utilisés pour exécuter les jeux. La console se synchronise automatiquement avec la bibliothèque Steam de l'utilisateur.

Installation et Mises à jour des Jeux

La structure de gestion des jeux permet de jouer même pendant les mises à jour :

      game/
      ├── instance/{nom_du_jeu}/      ← Version stable utilisée en cours
      └── update/{nom_du_jeu}/        ← Nouvelle version en cours de téléchargement

Quand une mise à jour est disponible, elle est téléchargée dans game/update/. Le jeu continue de fonctionner normalement depuis game/instance/. Une fois le téléchargement terminé, la console notifie le joueur qu’il pourra basculer vers la nouvelle version après avoir quitté sa session actuelle. Le système compare automatiquement les version.txt pour lancer la plus récente.
